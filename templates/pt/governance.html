Sistema de Governança Descentralizada

Governança descentralizada por Blockchain, ou DGBB, é a tentativa da Dash para resolver dois problemas importantes na criptocorrência: governança e financiamento.

O que é DGBB Tools

Governança em um projeto descentralizado é difícil, Porque, por definição, não há autoridades centrais para tomar decisões para o projeto. Na Dash, tais decisões são tomadas pela rede, ou seja, pelos proprietários de masternodes. O sistema DGBB permite os masternodes Votar a favor ou contra as propostas, que podem então ser implementadas (ou não) pelos desenvolvedores da Dash. Um exemplo no início de 2016, Quando a Core Team da Dash enviou uma proposta à rede perguntando se o tamanho do bloco deveria ser aumentado para 2 MB. Dentro de 24 horas, Foi alcançado um consenso para aprovar esta mudança. Compare isso com o Bitcoin, onde o debate sobre o tamanho de blocos vem se intensificando há quase três anos.


A DGBB também fornece um meio para a Dash financiar seu próprio desenvolvimento. Enquanto outros projetos têm de depender de doações Dash usa 10% da recompensa do bloco para financiar seu próprio desenvolvimento. Cada vez que um bloco é extraído, 45% da recompensa vai para o minerador, 45% vai para um Masternode, e o restante 10% não é criado até o final do mês. Durante o mês, qualquer pessoa pode fazer uma proposta de orçamento para a rede. Se essa proposta for aprovada em pelo menos 10% da rede masternode, então no final do mês será criada uma série de "superblocos". Nessa altura, as recompensas em bloco que não foram pagas (10% de cada bloco) serão utilizadas para financiar propostas aprovadas. A rede financia assim Reservando 10% do prêmio do bloco para projetos aprovados.

Votação automatizada

Um dos maiores desafios da construção de uma plataforma de criptomoeda é garantir que você criar um sistema descentralizado de governança para gerenciar, Financiar, manter e expandir o projeto. Este elemento chave está ausente em todas as moedas até a data, a resposta natural é criar uma organização sem fins lucrativos Que é encarregada de manter e promover a moeda, mas não está realmente ligado aos titulares de moedas de qualquer forma significativa. Esta abordagem tem algumas questões que foram tornadas a partir da experiência de plataformas de criptomoedas mais antigas.

Governança Tools

O sistema funcionaria como um mecanismo de votação descentralizado no software, onde os orçamentos para projetos específicos são propostos, Então os masternodes como um todo vota neles. Cada projeto, se for aprovado, é adicionado ao orçamento total e pago diretamente do blockchain Para a pessoa que faz o trabalho. Com isso, poderíamos contratar desenvolvedores principais e pagá-los diretamente após a aprovação do trabalho de forma descentralizada.
